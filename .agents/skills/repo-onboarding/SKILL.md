---
name: repo-onboarding
description: Explore an unfamiliar repository and produce a practical getting-started map.
---

When invoked:

1. Identify stack, package managers, build system, test system, and entry points.
2. Read README, package/config files, and main app entry files.
3. Summarize:
   - what the project does
   - main directories
   - how to run it
   - how to test it
   - key architectural patterns
   - where a new contributor should start
4. Highlight missing or outdated docs if found.

Rules:
- Be concrete.
- Prefer commands and file paths over generic explanations.
