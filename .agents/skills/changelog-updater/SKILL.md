---
name: changelog-updater
description: Turn recent code changes into clear release notes or changelog entries.
---

When invoked:

1. Inspect recent changes or diff context.
2. Separate user-visible changes from internal-only changes.
3. Draft changelog text in plain language.
4. Group into:
   - Added
   - Changed
   - Fixed
   - Deprecated (if relevant)
5. Keep technical jargon low unless the project audience is developers.

Rules:
- Focus on impact, not raw implementation detail.
- Do not include speculative claims.
