---
name: docs-from-code
description: Generate or improve documentation from the codebase, public APIs, and usage patterns.
---

When invoked:

1. Identify the audience:
   - end user
   - contributor
   - internal developer
2. Read code and examples before documenting.
3. Produce concise docs with:
   - purpose
   - setup or usage
   - examples
   - caveats
4. Prefer practical examples over abstract descriptions.

Rules:
- Do not document behavior not supported by the code.
- Call out unclear areas instead of inventing answers.
