---
name: prompt-to-plan
description: Turn a natural-language task into a concrete implementation plan with risks, files, and milestones.
---

When invoked:

Produce:
1. Goal
2. Assumptions
3. Likely files/modules involved
4. Step-by-step implementation plan
5. Risks/unknowns
6. Validation plan
7. Definition of done

Rules:
- Be concise but actionable.
- Prefer repo-specific paths and commands where possible.
